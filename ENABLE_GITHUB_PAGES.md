# Enable GitHub Pages - Quick Fix

Your site is deployed but GitHub Pages needs to be enabled in your repository settings.

## Steps to Enable GitHub Pages:

1. **Go to your repository on GitHub:**
   - Visit: https://github.com/telmon95/portfoliov2

2. **Click on "Settings"** (top menu of your repository)

3. **Scroll down to "Pages"** (left sidebar, under "Code and automation")

4. **Under "Source":**
   - Select: **"Deploy from a branch"**
   - Branch: Select **"gh-pages"**
   - Folder: Select **"/ (root)"**
   - Click **"Save"**

5. **Wait 2-5 minutes** for GitHub Pages to build and deploy

6. **Your site will be live at:**
   - https://telmon95.github.io/portfoliov2

## Troubleshooting:

- If you don't see "Pages" in settings, make sure the repository is **Public** (not Private)
- After enabling, wait a few minutes and refresh the page
- Check the "Actions" tab to see if there are any build errors
